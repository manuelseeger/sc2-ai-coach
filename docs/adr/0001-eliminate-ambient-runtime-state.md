# Eliminate Ambient Runtime State Via Explicit Settings Loading And Composition

The project will remove ambient runtime state such as import-time config loading, module-level convenience accessors, process-global infrastructure singletons, package import-time implementation selection, and import-time logging setup. Runtime settings are loaded only through an explicit loader API that returns a fresh settings object per call or raises a project-specific loader error; a shared composition module then derives module-owned configuration, performs implementation selection, wires infrastructure explicitly, and creates configured objects only inside execution paths used by `coach`, `repcli`, the future API, and tests. This was chosen over lazy globals or compatibility shims so those entrypoints remain import-safe by default while tests still retain an explicit opt-in path for loading current yaml/env-backed settings.
